# 一期新增：文章标签生成模块

## 一、目标

一期需要为每篇导入文章自动生成 8–10 个文章标签，类似小红书关键词，用于文章展示、检索、知识库分类和后续推荐。

标签需要兼顾：

1. 用户能看懂。
2. 适合搜索。
3. 适合知识库分类。
4. 适合 QA 召回。
5. 不夸大、不营销、不制造焦虑。

---

```yaml

    min: 8

  style:

    - "患者容易理解"
    - "适合搜索"
    - "避免标题党"
    - "避免夸大疗效"
```

## 五、展示型标签生成规则

展示型标签 display_tags 需要类似小红书关键词，但风格要克制、专业。

要求：

```text
1. 每篇文章生成 8–10 个。
2. 每个标签 2–8 个汉字为佳。
3. 优先使用患者真实会搜索的词。
4. 优先覆盖疾病、主题、场景、对象和问题。
5. 不要生成过度营销标签。
6. 不要生成“神药”“治愈”“救命疗法”“逆转癌症”等夸大词。
7. 医学治疗类标签要保持中性。
8. 心理、营养、护理类内容要避免制造焦虑。
```

## 七、标签生成逻辑

Agent 处理流程增加一步：

```text
输入文章
  ↓
识别文章类型和领域
  ↓
提取主题标签
  ↓
生成 display_tags 8–10 个
  ↓
生成 structured_tags
  ↓
生成 QA
  ↓
评分
  ↓
输出 JSON
```

---

## 八、标签质量评分

一期不单独给标签打复杂分，但可以在 QA 可用性中考虑标签质量。

如果标签清楚、准确、适合搜索，可提高 QA 可用性评分。
如果标签过泛、过营销、和正文不匹配，需要扣分。

扣分规则：

```yaml
tag_deduction_rules:
  too_generic_tags:
    condition: "标签过泛，例如健康、科普、癌症、治疗等占比过高"
    penalty: 3

  misleading_tags:
    condition: "标签与正文主题不匹配"
    penalty: 5

  marketing_tags:
    condition: "标签包含神药、治愈、逆转、奇迹、救命等营销词"
    penalty: 8

  insufficient_tags:
    condition: "标签少于8个"
    penalty: 3
```

## 十、一期开发要求

1. 每篇文章必须输出 display_tags。
2. display_tags 数量为 8–10 个。
3. 每篇文章必须输出 structured_tags。
4. 标签必须来自正文主题，不要凭空扩展。
5. 标签要适合患者搜索和前端展示。
6. 标签不能标题党化。
7. 标签不能包含夸大治疗效果的词。
8. 标签质量可以影响 QA 可用性评分。

核心流程：人工精选文章 → 自动整理 QA → 生成文章标签 → 打 QA 分数 → 分级入库
